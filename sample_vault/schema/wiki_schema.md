# Wiki Schema

This document defines the structure of all wiki pages and metadata files.

## Vault Directory Layout

```
vault/
├── raw/                    # Immutable source files (chmod 444)
│   ├── {source-id}.{ext}   # Original file
│   └── {source-id}.meta.json
├── normalized/             # Markitdown-converted markdown
│   └── {source-id}.md
├── wiki/
│   ├── index.md            # Master table of contents
│   ├── log.md              # Chronological activity log
│   ├── overview.md         # Auto-generated wiki summary
│   ├── sources/            # One page per ingested source
│   ├── entities/           # People, orgs, products, places, events
│   ├── concepts/           # Technical terms, theories, methods
│   ├── topics/             # Broad subject areas
│   ├── analyses/           # Q&A saved from `ask --save`
│   └── reports/            # Custom reports
└── schema/
    ├── AGENTS.md           # Agent behavior instructions
    ├── wiki_schema.md      # This file
    └── page_conventions.md # Formatting conventions
```

## Source Metadata (`.meta.json`)

```json
{
  "source_id": "string",
  "original_name": "string",
  "original_path": "string",
  "extension": "string",
  "size_bytes": 0,
  "content_hash": "string (sha256)",
  "added_at": "ISO8601Z",
  "normalized_at": "ISO8601Z | null",
  "ingested_at": "ISO8601Z | null",
  "title": "string | null",
  "tags": ["string"]
}
```

## Source Page Schema (`wiki/sources/{source-id}.md`)

```yaml
---
source_id: string
title: string
type: source
added: YYYY-MM-DD
date_published: YYYY-MM-DD | null
authors: [string]
tags: [string]
---
```

Required sections:
- `## Summary` — 2-4 sentence overview
- `## Key Points` — bullet list of main findings/arguments
- `## Entities` — links to entity pages
- `## Concepts` — links to concept pages
- `## Topics` — links to topic pages
- `## Source` — links to raw and normalized files

## Entity Page Schema (`wiki/entities/{slug}.md`)

```yaml
---
name: string
type: entity
entity_type: person | organization | product | place | event | other
slug: string
---
```

Required sections:
- `## Description` — what this entity is
- `## Sources` — citations (with quotes where relevant)

Optional sections:
- `## Related Entities`
- `## Related Concepts`

## Concept Page Schema (`wiki/concepts/{slug}.md`)

```yaml
---
name: string
type: concept
slug: string
---
```

Required sections:
- `## Description` — plain-language explanation
- `## Sources` — citations

Optional sections:
- `## Related Concepts`
- `## Examples`

## Topic Page Schema (`wiki/topics/{slug}.md`)

```yaml
---
name: string
type: topic
slug: string
---
```

Required sections:
- `## Overview` — broad description of the topic
- `## Sources` — list of sources covering this topic

Optional sections:
- `## Related Topics`
- `## Key Concepts`

## Analysis Page Schema (`wiki/analyses/{date}-{slug}.md`)

```yaml
---
type: analysis
question: string
answered_at: ISO8601Z
confidence: high | medium | low
sources_consulted: [string]
---
```

## Index Format (`wiki/index.md`)

The index is a plain markdown file with sections:

```markdown
# Wiki Index

## Sources
- [{source-id}](sources/{source-id}.md) — {title} | tags: {tags} | ingested: {date}

## Entities
- [{slug}](entities/{slug}.md) — {name} ({entity_type})

## Concepts
- [{slug}](concepts/{slug}.md) — {name}

## Topics
- [{slug}](topics/{slug}.md) — {name}
```

## Log Format (`wiki/log.md`)

```markdown
# Wiki Log

## {ISO8601Z timestamp}
- {action description}
- {page created/updated}
```

## Slug Generation

Slugs are generated by:
1. Lowercasing the name
2. Removing all non-word characters except spaces and hyphens
3. Replacing whitespace and underscores with hyphens
4. Collapsing multiple hyphens
5. Trimming leading/trailing hyphens

Example: `"Attention Is All You Need!"` → `"attention-is-all-you-need"`

## Source ID Generation

Source IDs combine a slug from the filename with an 8-character SHA-256 prefix of the file content:

`{filename-slug}-{8-char-hash}`

Example: `intro-to-transformers-a1b2c3d4`
